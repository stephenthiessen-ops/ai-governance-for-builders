# AI Governance Operating Model

AI governance must balance velocity and risk control.

---

## Governance Layers

### 1. Policy Layer
Defines what is permitted and prohibited.
- Approved model providers
- Data classification rules
- External exposure restrictions

### 2. Control Layer
Enforceable mechanisms:
- Prompt versioning
- Access controls
- Logging
- Review gates

### 3. Review Layer
- Risk tier classification
- Periodic system review
- Escalation path

### 4. Audit Layer
- Input traceability
- Output archiving
- Change history

---

## Governance Principles

- Deterministic systems before generative interpretation
- Clear accountability ownership
- Risk-tier-based enforcement
- Human oversight for high-impact decisions
